# 企业级工单智能分流与协同处理系统（简历项目描述）

**角色**：独立开发（AI 模块 + 后端 + 前端）  ｜  **技术栈**：Java 8 / Spring Boot 2.7 / Flowable 6.8 / Redis(Redisson) / Vue3+Vite / MySQL / Docker

## 项目概述
面向企业的工单系统：用 AI 对自然语言工单做语义分类与智能路由，按业务规则驱动 Flowable 工作流协同处理，解决人工分派慢、易错、审批链路不统一的问题。

## 核心工作
- 自研 **TF-IDF 语义向量 + 少样本检索投票分类器**，构建 54 条**严格留出**的人工标注评测集；运行语料为 **36 条独立种子**（与评测集 0 文本重叠、无数据泄漏），独立评测准确率 **98.1%（53/54）**、泛化 90% 且 HIL 自动误派率 0%，替代原关键词匹配方案。
- 设计**置信度阈值触发的人工复核闭环（HIL）**：低置信工单不自动启动流程、进复核队列，人工确认/改判经在线学习回流向量库，实测**拦截全部误分流**。
- 抽象 `Function<String,Map<String,Double>>` 向量化接口，**零改动即可从本地 TF-IDF 切换为 BGE/OpenAI 稠密语义 embedding**（OpenAI 兼容 `/v1/embeddings`，L2 归一化 + LRU 缓存），附单测证明可插拔性。
- 实现**主动学习数据飞轮**：低置信样本自动入池、人工标注经 `learn()` 在线回流，复评准确率提升、HIL 触发降为 0，且对评测集外相似句具备泛化。
- 基于 Flowable 实现**条件性路由**：AI 分类决定流程是否含财务打款节点（非前端隐藏）；集成 Redisson 分布式锁保证建单 / Claim 幂等。
- **工程化交付**：Docker Compose 编排 MySQL+Redis+应用（健康检查依赖启动）、springdoc 自动接口文档、前端产物打进单 jar 同源托管。

## 技术亮点（面试可深挖）
- 语义检索分类为何优于关键词匹配 / 模型微调；余弦相似度局限与稠密 embedding 价值
- 置信度校准 `confidence = voteShare × min(1, topSim/0.35)` 防假阳性
- HIL 阈值（0.55）依据评测集置信度分布与误分流率反推
- 主动学习 uncertainty sampling 采样策略与冷启动兜底（种子语料 + 规则 + 人工复核）

## 成果
- 端到端回归 **59/59 PASS**；独立评测（运行语料与评测集 0 重叠）分类准确率 **98.1%（53/54）**、泛化 **90%**、HIL 自动误派率 **0%**；单 jar 同源托管，Docker 一键部署可演示。
